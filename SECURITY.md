# Security Policy

## Reporting a vulnerability

Please report security issues privately by emailing **daniel.siegle@gmail.com**.
Do not open a public issue for security-sensitive reports.

Include a description of the issue, steps to reproduce, and the affected
version. We aim to acknowledge reports within a few business days and will keep
you informed as we work on a fix.

## Supported versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |
